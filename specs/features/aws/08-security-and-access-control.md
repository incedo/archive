# AWS Stories - Feature 08 Security and Access Control

Feature source: [08-security-and-access-control.md](/Users/kees/data/projects/archive/features/08-security-and-access-control.md)

## AWS components

- IAM Identity Center
- IAM
- KMS
- API authorization layer

## Stories

- `AWS-08-01` Authentication integration for workforce users and workloads.
- `AWS-08-02` Role-based authorization for finance, legal, audit and operations.
- `AWS-08-03` Scope-based authorization by legal entity, document type and confidentiality class.
- `AWS-08-04` Encryption in transit and managed-key usage for storage and APIs.
- `AWS-08-05` Fine-grained rights for read, export, metadata update, disposition and legal hold.
- `AWS-08-06` Extra protection paths for sensitive records and high-impact admin actions.

## Acceptance focus

- authorization decisions are explainable
- encryption is baseline, not the only control
- high-risk actions have stronger checks
