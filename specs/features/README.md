# Stories

## Doel

Deze map vertaalt de cloud-onafhankelijke features naar provider-specifieke implementatiestories. 
In de root staan de onafhankelijke features, in de sub mappen per provider staan de concequenties


De structuur is:

- [aws](/Users/kees/data/projects/archive/stories/platforms/aws)
- [scaleway](/Users/kees/data/projects/archive/stories/platforms/scaleway)

Per provider geldt:

- een featurebestand per hoofdfeature
- stories zijn nog steeds solution-level, niet sprint-task-level
- verdere opsplitsing naar kleinere engineering stories kan later per team of iteratie

## Gebruik

Werk stories uit in deze volgorde:

1. featurebestand kiezen
2. benodigde skills bepalen via [CODEX_SKILLS_REQUIRED.md](/Users/kees/data/projects/archive/CODEX_SKILLS_REQUIRED.md) en [SKILL_TO_FEATURE_MAPPING.md](/Users/kees/data/projects/archive/SKILL_TO_FEATURE_MAPPING.md)
3. relevante story selecteren
4. acceptance criteria aanscherpen voor de iteratie
5. implementeren volgens de [RALPH_WIGGUM_LOOP_PLAN.md](/Users/kees/data/projects/archive/RALPH_WIGGUM_LOOP_PLAN.md)
